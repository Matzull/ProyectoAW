Concepto

Bienvenido a nuestra revolucionaria aplicación de computación distribuida, Parallelize. ¿Alguna vez has querido procesar un programa, pero tu ordenador no tiene suficiente capacidad para hacerlo?

Nuestra aplicación es una plataforma que permite a las personas compartir el poder de cómputo de sus ordenadores. Esto puede ser útil para tareas que requieren mucha potencia de procesamiento, como hacer grandes cálculos matemáticos o entrenar redes neuronales no profesionales.

Los usuarios pueden ofrecer su ordenador para el procesamiento de programas y recibir créditos a cambio, o simplemente comprarlos para poder hacer uso del poder de la red. Los créditos obtenidos pueden ser retirados o reinvertidos para el procesamiento de nuevos programas.

Hay 3 tipos de Usuario

Invitado

  • Iniciar sesión y registrarse en la web.

Usuario registrado

  • Subir kernels propios.
  • Ejecutar kernels de la comunidad.
  • Introducir y retirar dinero en forma de créditos.
  • Denunciar a un usuario si encuentra un kernel malicioso.
  • Ver el ranking de participación.
  • Ver la estimación de la complejidad al ejecutar un kernel subido por otro usuario.
  • Aportar una valoracion tras ejecutar el kernel subido por otro usuario.

Administrador

El administrador, además de tener las funciones del usuario registrado, tendrá una serie de permisos adicionales como son:

  • Cambiar el timeout de avisos de inactividad del kernels.
  • Banear a usuarios.
  • Administrar el ranking.
  • Juzgar las denuncias.

Tareas de la web

  • Registrar a un usuario nuevo.
  • Permitir a un usuario acceder a su cuenta.
  • Gestionar el retiro e ingreso de créditos de un usuario.
  • Subir a la red el kernel de un usuario.
  • Proporcionar el kernel a un usuario para que lo ejecute.
  • Evaluar la complejidad de los kernels para realizar una estimación de la complejidad.
  • Calcular un ranking de participación.
  • Utilizar un sistema de banHTML donde los administradores podrán recibir las denuncias de los usuarios y podrán, si lo consideran justo, deshabilitar la cuenta a un usuario.
  • Tener implementado un sistema de valoración para cada kernel que suba un usuario.

Funcionalidades

  1. Gestión de Administrador.
  2. Gestión de ranking.
  3. Gestión de ejecución de kernels (incluye la gestión de los puntos o créditos obtenidos).
  4. Gestión de subida de kernels (incluye la gestión de la evaluación de la complejidad).
  5. Gestión de valoraciones (incluye la gestión de las valoraciones de los usuarios que suben el kernel y la valoración de los usuarios que los ejecutan).
  6. Gestión de promociones de código ó gestión de retirada de dinero en forma de créditos.
